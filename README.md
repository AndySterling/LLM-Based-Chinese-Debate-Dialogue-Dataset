# LLM-Based 中文论辩对话数据集

## 1. 数据集介绍
这是一个基于知乎、微博平台用户交互数据，经过数据清洗和大语言模型（Large language model, LLM）数据扩充得到的中文论辩对话数据集。<br>
具体而言，先利用网络爬虫根据特定主题从知乎、微博平台获取相应的数据，然后清洗掉一些低质数据（如长度过短、包含情绪化词语等），再基于预先设定的人工标注模板，驱动LLM对数据进行标注，最后基于标注结果驱动LLM生成多轮论辩对话。<br>
<br>

**贡献:**<br>
现有公开数据集大多面向立场检测或论证成分识别，数据形式多为片段化的单轮论证或短评论链，缺少贴近日常口语交流且带有胜负结局的多轮论辩对话。Zhao等人构建的[OrChiD数据集][OrChiD]，虽然也属于中文领域论辩数据集，但其数据来源于正式辩论比赛，其语料逻辑严谨，但是语言偏正式且格式固定，与日常网络辩论中口语化、碎片化和立场快速交替的特点存在较大差异。**为解决中文领域缺少贴近日常网络辩论对话数据的问题，本数据集模拟网民日常辩论风格，提供了口语化、碎片化的多轮论辩对话数据。**<br>
<br>

**使用本数据集须知：数据集内容基于真实人类交互数据，但由于LLM存在幻觉现象，请仔细检查内容是否存在虚构！此外，本数据集已尽量去除情绪化发言内容**<br>
**建议从论辩策略方面开展研究，不建议使用该数据集使模型学习世界知识**

## 2. 字段结构
下面展示了数据的字段结构：<br>
>{"topic": "xxx", "winner": null, "rounds": [{"speaker": "A", "content": "A said sth."}, {"speaker": "B", "content": "B said sth."}, ...], "seed_id": "id_example", "auto_annotation": {"数据类型": "评论链", "原文摘要": "yyy", "立场倾向": "反对", "辩论价值评分": 4, "核心论点": "argument_example", "论证结构": "因果推理；对比分析；归因分析；逻辑链", "可辩论性": "高", "备注": "[用户立场]反对[用户观点]user_view_example[备注说明]评论链从微观到宏观层层递进，论证结构清晰，但缺乏具体数据支撑。", "数据ID": "id_example"}}<br>

其中：<br>
**topic：** 论辩主题<br>
**winner：** 论辩胜方，可能的取值为“A”、“B”、“null”（僵持型对话，没有胜方）<br>
**rounds：** 论辩对话轮次列表，每个元素为字典对象，其包含两个字段：<br>
    &emsp;&emsp;&emsp;&emsp;**speaker：** 发言方，可能的取值为“A”、“B”<br>
    &emsp;&emsp;&emsp;&emsp;**content：** 发言内容<br>
**seed_id：** 论辩对话数据对应的种子数据（即其对应的微博/知乎数据）id<br>
**auto_annotation：** LLM自动标注内容，字典对象，其包含以下字段：<br>
    &emsp;&emsp;&emsp;&emsp;**数据类型：** 可能的取值为“回答”、“评论链”<br>
    &emsp;&emsp;&emsp;&emsp;**原文摘要：** 种子数据内容摘要<br>
    &emsp;&emsp;&emsp;&emsp;**立场倾向：** 种子数据用户的立场，可能的取值为“支持”、“反对”、“混合”（可能为用户部分支持某一观点；或数据为评论链，交互双方持相反立场）<br>
    &emsp;&emsp;&emsp;&emsp;**辩论价值评分：** 对种子数据的论证质量、信息量、可辩论性的综合评分，取值范围为1-5<br>
    &emsp;&emsp;&emsp;&emsp;**核心论点：** 辩论的焦点内容<br>
    &emsp;&emsp;&emsp;&emsp;**论证结构：** 数据中使用的论辩结构，如“因果分析”等<br>
    &emsp;&emsp;&emsp;&emsp;**可辩论性：** 衡量论点是否容易引发有深度的、多回合的反驳与辩护，取值范围为“高”、“中”<br>
    &emsp;&emsp;&emsp;&emsp;**备注：** 其结构为“[用户立场]立场信息，与‘立场倾向’字段取值相同[用户观点]用户观点信息，若‘立场倾向’字段为‘混合’，则包含正反立场，格式为‘正方：正方观点；反方：反方观点’；若‘立场倾向’字段为‘支持’或‘反对’，则与‘核心论点’字段相同[备注说明]对数据的评价和说明”<br>
    &emsp;&emsp;&emsp;&emsp;**数据ID：** 论辩对话数据对应的种子数据（即其对应的微博/知乎数据）id，与“seed_id”字段相同<br>
<br>
